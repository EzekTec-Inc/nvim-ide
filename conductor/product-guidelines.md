# Product Guidelines

## Naming & Structuring Conventions
- **Snake_case:** All new Lua modules, plugin files, and configuration scripts must follow standard Neovim/Lua `snake_case` naming conventions (e.g., `plugin_name.lua`). Consistency with the broader Neovim ecosystem is paramount.

## Documentation Style
- **Inline Comments:** Prioritize concise inline comments that explain the *why* behind configuration choices, keeping the code itself clean and self-documenting.
- **Extensive READMEs:** Each major configuration module or complex plugin setup should be accompanied by its own dedicated markdown file to provide deeper context, usage instructions, and troubleshooting tips.

## UI & Visual Identity
- **Minimalist:** Design interfaces to maximize screen space for code. UI elements should be hidden until needed, reducing cognitive load and distractions.
- **Aesthetic Focus:** Maintain a strong focus on a cohesive visual experience using custom themes, devicons, and consistent styling across all editor components to create an inspiring developer environment.
