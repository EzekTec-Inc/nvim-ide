# Restore Point: nvim-ufo E1511 Fix

**Date:** 2026-02-13T23:18:27-07:00 (Friday)

## Error Fixed

```
E1511: Wrong number of characters for field "foldopen"
```

## Root Cause

The E1511 error occurs when Neovim validates fillchars containing `foldopen`, `foldsep`, or `foldclose` fields. These fields require exactly one single-byte ASCII character. Empty strings or multi-byte Unicode characters are invalid.

The error was triggered because:
1. Somewhere in the config initialization, fillchars were being set or validated
2. The foldcolumn setting and fold options ordering was causing a race condition

## Fix Applied

1. **`init.lua`** (nvim-ufo init function):
   - Reordered fold option settings to set `foldcolumn = "0"` first
   - Set `foldmethod = "manual"` immediately after to ensure no fold operations trigger before options are set

2. **`lua/options.lua`**:
   - Changed from `vim.opt.fillchars:append({ eob = " " })` to `vim.o.fillchars = "eob: ,fold: "`
   - Using string format avoids any table parsing issues
   - Only sets `eob` and `fold` fields, NOT `foldopen`/`foldsep`/`foldclose`

## To Restore Previous State

If this fix causes issues:

```lua
-- In init.lua nvim-ufo init function, change back to:

  vim.o.foldenable = true

end,

-- In lua/options.lua, change back to:

```
