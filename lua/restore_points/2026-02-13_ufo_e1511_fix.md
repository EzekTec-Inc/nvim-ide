# Restore Point: nvim-ufo E1511 Fix

**Date:** 2026-02-13T23:17:15-07:00 (Friday)

## Error Fixed

```
E1511: Wrong number of characters for field "foldopen"
```

## Root Cause

The nvim-ufo plugin config function was setting `vim.o.foldcolumn = "0"` inside the `config` function, but the fold options need to be set BEFORE the plugin loads to prevent Neovim from validating fillchars for fold-related fields.

## Fix Applied

Moved fold-related vim options (`foldcolumn`, `foldenable`, `foldlevel`, `foldlevelstart`) from the `config` function to an `init` function in the nvim-ufo plugin spec. The `init` function runs before the plugin loads, ensuring these options are set before any fold operations occur.

## Files Modified

- `init.lua` - Changed nvim-ufo plugin spec to use `init` function for fold options

## To Restore Previous State

If you need to revert this change:

```lua
-- In init.lua, change the nvim-ufo plugin spec from:
    init = function()
      -- FIX 2026-02-13T23:17:15: Set fold options BEFORE plugin loads
      -- E1511 error occurs when foldcolumn > 0 and fillchars foldopen/foldclose are invalid
      -- Setting these in init (not config) ensures they're set before any fold operations
      vim.o.foldcolumn = "0"
      vim.o.foldenable = true
      vim.o.foldlevel = 99
      vim.o.foldlevelstart = 99
    end,
    config = function()

-- Back to:
    config = function()
      -- FIX 2026-02-13T23:16:48: Disable native foldcolumn to prevent E1511 error
      -- The E1511 error occurs when fillchars validation runs for fold-related fields
      -- Setting foldcolumn to "0" prevents neovim from requiring foldopen/foldsep/foldclose
      vim.o.foldcolumn = "0"
      vim.o.foldenable = true
      vim.o.foldlevel = 99
      vim.o.foldlevelstart = 99
```
